# novelWebTool — AI 小说写作助手

一个运行在浏览器中的 **AI 辅助小说创作工具**，纯前端架构，数据存储在本地 IndexedDB，通过调用大模型 API 实现从故事构思到正文写作的完整创作流程。

---

## 功能特性

### 📚 小说管理
- 新建、编辑、删除小说
- 支持分类标签（玄幻、言情、悬疑等）

### 📝 章节管理
- 创建/编辑/删除章节
- 批量创建章节、调整章节顺序、合并章节
- 章节简述（大纲）与正文分离管理

### 👤 人物设定
- 创建角色（姓名、性别、年龄、性格、外貌、背景）
- 人物关系图（可视化连线，支持单向/双向关系）
- 人物关系自动维护

### 🌍 世界观设定
- 自由创建世界观条目（世界规则、地理、历史等）

### ⏳ 时间线
- 多轨道时间线（如主线、感情线、支线）
- 事件按序排列，支持重点标记和跨段事件

### 📄 故事大纲
- 编辑总大纲，全局把控剧情走向

### 🤖 AI 辅助写作
- 右侧聊天面板，AI 能理解小说上下文
- Function Calling 机制：AI 可直接查询/创建/修改素材
- 支持流式输出（SSE）
- 可配置多种 API（DeepSeek、OpenAI、通义千问等）
- 小说记忆摘要（AI 生成的持久化全局理解，跨对话保留）

### 📎 附件选择
- 跟 AI 对话时可选投喂的上下文（人物、世界观、大纲、前文等）

### 📤 导入/导出
- 整部小说的 JSON 导入导出

### 🌙 暗色模式
- 亮色/暗色一键切换

## 技术栈

| 技术 | 用途 |
|------|------|
| **Vue 3** | 前端框架 |
| **TypeScript** | 类型安全 |
| **Vite** | 构建工具 |
| **Pinia** | 状态管理 |
| **Vue Router** | 路由 |
| **Element Plus** | UI 组件库 |
| **IndexedDB** | 本地数据存储 |
| **OpenAI 兼容 API** | AI 对话与工具调用 |

## 项目结构

```
src/
├── components/
│   ├── ai/          # AI 交互组件（问题卡片、方案卡片等）
│   ├── chapter/     # 章节管理组件
│   ├── character/   # 人物设定组件
│   ├── chat/        # AI 对话面板组件
│   ├── common/      # 通用组件
│   ├── dialogs/     # 弹窗组件
│   └── world/       # 世界观/时间线组件
├── composables/     # 组合式逻辑
├── config/          # 配置文件（API 预设、系统提示词）
├── stores/          # Pinia 状态仓库
├── types/           # TypeScript 类型定义
├── utils/
│   ├── ai/          # AI 工具定义、处理器、流式请求
│   ├── db.ts        # IndexedDB 封装
│   ├── prompt.ts    # AI 提示词构建
│   └── id.ts        # ID 生成器
└── views/           # 页面组件
```

## 快速开始

```bash
# 安装依赖
npm install

# 启动开发服务器（热更新）
npm run dev

# 构建生产版本
npm run build
```

## 使用说明

1. **配置 API**：点击右上角"API 设置"，填入 API 地址和 Key
2. **新建小说**：设置小说名称、类型和简介
3. **完善设定**：在小说详情页中管理人物、世界观、时间线
4. **AI 写作**：在右侧聊天面板输入需求，AI 会根据现有素材辅助创作
